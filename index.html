    Bethesda Modding Library

    A somewhat ideosyncratic reference library for modding Skyrim (LE/SE), Fallout 4, and Starfield, focusing on file formats and Python tools. If you work with AI, hand this library to the bot as a resource so it doesn't have to reconstruct all this itself.

    NOT ALL THIS INFO IS VETTED BY AN ACTUAL HUMAN. Where I've reviewed and validated a page, there's a timestamp at the bottom — which doesn't guarantee it's correct, just that I looked at it. Corrections welcome.

    LOTS OF ELEMENTS ARE STILL TBD. I'm building this as I need it, so many pages aren't written yet. It's more complete for Starfield and FO4 than for Skyrim, because that's where I'm working right now. Links tagged [TBD] point at pages that don't exist yet — no need to chase them.

    What's Inside

    This library consolidates knowledge from multiple sources into a unified, searchable reference for Bethesda game modding. Whether you're creating new armor, custom races, or complex animations, you'll find the technical details you need here.

    📁 File Formats

    Deep dives into Bethesda's binary file formats (shared across Skyrim and Fallout 4):

    • Plugins (ESP/ESM) - Record structure, FormIDs, master resolution
    • NIF Files - Meshes, skinning, shaders, partitions
    • NIF Animations - Scene-graph object animations (doors, banners, glow)
    • Animations (HKX) - Havok skeleton and animation files
    • Physics & Collision - Havok physics, HDT-SMP configs
    • Morphs & Shape Keys - TRI files, expressions, chargen
    • Textures & Materials - DDS, BGSM/BGEM
    • Archives (BSA) - BSA format
    • Scripts (PEX) - Compiled Papyrus

    🎮 Game-Specific Information

    Where a game diverges from the shared formats above, it gets its own section fronted by a game-overview page:

    • Starfield - Creation Engine 2: geometry-less NIF + external .mesh, layered .mat/.cdb materials, component-based plugins, in-house animation. The most complete area right now:
    • Meshes (NIF + .mesh) - BSGeometry, external .mesh binary format, skinning
    • Materials & Textures - layered .mat/.cdb, shader models, DDS conventions (worked example)
    • Chargen, Race & Skeleton - RACE/HDPT/MRPH/BMOD, morphs, custom races
    • Plugins & Archives - component records, master tiers, BA2 v2/v3
    • Tools - the Starfield tooling directory, grouped by task
    • Fallout 4 - 64-bit Creation Engine: BGSM/BGEM materials, mesh segments, connect points, BA2 archives, ESL plugins:
    • Connect Points - runtime mesh attachment points
    • Dismemberment - limb severing (BSSubIndexTriShape segments, .ssf)
    • Skyrim - LE (32-bit, NiTriShape) and SE (64-bit, BSTriShape); shared formats with per-edition differences

    🐍 Repository Scripts

    Python scripts that live in the repository — used while working out the plugin format. Most are Skyrim-focused investigation scripts with hardcoded data paths, so read them before running:

    • dump_hkx.py - dump a Havok .hkx packfile's structure as human-readable text (the one general-purpose tool here)
    • inspect_masters.py - dump a plugin's TES4 header and master list
    • FormID range investigation:
    • check_all_formid_ranges.py - FormID ranges across masters, DLC, and ESL files
    • check_formids_deep.py - recurse records and FormIDs in Skyrim.esm
    • check_formids_raw.py - first 50 FormIDs in Skyrim.esm
    • check_esl_formids.py - ESL-relevant FormID ranges in Skyrim.esm + DLCs
    • check_esl_ranges.py - FormID ranges in actual .esl files
    • check_dawnguard.py - recursive record read of Dawnguard.esm
    • Override detection:
    • check_overrides.py - read subrecords and detect override records
    • check_override_final.py - override detection with record-type filtering
    • simple_override_check.py - examine overrides in a single ESP

    Scope

    Games Covered: - ✅ Skyrim LE (32-bit) - ✅ Skyrim SE (64-bit) - ✅ Fallout 4 (64-bit) - ✅ Starfield (Creation Engine 2) - most complete area right now

    Tools Covered: - ✅ PyNifly (Blender addon, Python API) - ✅ esplib (Python library) - ✅ xEdit (reference only) - ✅ Creation Kit (brief reference)

    Explicitly Out of Scope: - ❌ Fallout 76, Fallout 3, Fallout New Vegas - ❌ Pascal/xEdit scripting (deprecated) - ❌ Detailed Creation Kit workflows - ❌ Executable validators or decision tree functions

    Organization Principles

    • Format vs Tool - File format specs are tool-agnostic; tool-specific conventions are clearly marked
    • Universal vs Game-Specific - Common information in "File Formats"; per-game divergences in "Game-Specific"
    • Blender-Agnostic - Game concepts explained first, then Blender implementation details
    • Complete Examples - No placeholders or "fill in" comments in code examples
